# 书籍缓存机制

## 概述

阅读应用在使用书源阅读网络书籍时，会自动缓存章节内容到本地，以实现离线阅读和提升阅读体验。本文档详细分析了缓存的存储位置、文件格式、清理机制以及备份可行性。

## 一、缓存是否开启

**是的，阅读网络书源书籍时会自动缓存。**

### 自动缓存机制

1. **阅读时自动缓存**
   - 当前章节、上一章、下一章的内容会自动缓存
   - 支持预下载功能，可根据设置预下载前后若干章节

2. **手动缓存**
   - 可在书籍详情页或目录页手动缓存整本书
   - 支持批量缓存多本书籍

### 相关代码位置

- 缓存服务：[CacheBookService.kt](../app/src/main/java/io/legado/app/service/CacheBookService.kt)
- 缓存管理：[BookHelp.kt](../app/src/main/java/io/legado/app/help/book/BookHelp.kt)
- 阅读加载：[ReadBook.kt](../app/src/main/java/io/legado/app/model/ReadBook.kt)

## 二、缓存位置

### 存储路径

缓存文件存储在应用的外部存储目录下：

```
/storage/emulated/0/Android/data/{包名}/files/book_cache/
```

对于 Legado Plus：
```
/storage/emulated/0/Android/data/io.legado.plus/files/book_cache/
```

### 代码定义

在 `BookHelp.kt` 中定义：

```kotlin
private val downloadDir: File = appCtx.externalFiles
private const val cacheFolderName = "book_cache"

val cachePath = FileUtils.getPath(downloadDir, cacheFolderName)
```

### 目录结构

```
book_cache/
├── 书名_作者/                    # 每本书一个文件夹
│   ├── 00001-a1b2c3d4e5f6g7h8.nb    # 章节内容文件
│   ├── 00002-b2c3d4e5f6g7h8i9.nb
│   ├── 00003-c3d4e5f6g7h8i9j0.nb
│   └── images/                       # 图片缓存（漫画类书籍）
│       ├── a1b2c3d4e5f6g7h8.jpg
│       └── b2c3d4e5f6g7h8i9.png
│
└── 另一本书_作者/
    ├── 00001-d4e5f6g7h8i9j0k1.nb
    └── ...
```

## 三、文件保存格式

### 章节内容文件

**文件命名格式：**
```
{章节序号}-{标题MD5}.nb
```

**示例：**
```
00001-a1b2c3d4e5f6g7h8.nb    # 第1章
00002-b2c3d4e5f6g7h8i9.nb    # 第2章
00123-b2c3d4e5f6g7h8i9.nb    # 第123章
```

**命名规则说明：**

| 组成部分 | 说明 |
|----------|------|
| 章节序号 | 5位数字，前面补0（如 `00001`、`00123`） |
| 标题MD5 | 章节标题的16位MD5编码，避免文件名过长或包含特殊字符 |
| 扩展名 | `.nb`（默认扩展名） |

### 代码实现

在 `BookChapter.kt` 中定义文件名生成：

```kotlin
@SuppressLint("DefaultLocale")
fun getFileName(suffix: String = "nb"): String {
    ensureTitleMD5Init()
    return String.format("%05d-%s.%s", index, titleMD5, suffix)
}
```

### 图片文件（漫画类）

对于漫画类书籍，图片文件命名格式：

```
{图片URL的MD5}.{图片后缀}
```

例如：
```
a1b2c3d4e5f6g7h8.jpg
b2c3d4e5f6g7h8i9.png
```

### 保存方式

章节内容使用纯文本文件保存，直接调用 Kotlin 的 `writeText()` 方法：

```kotlin
fun saveText(book: Book, bookChapter: BookChapter, content: String) {
    if (content.isEmpty()) return
    FileUtils.createFileIfNotExist(
        downloadDir,
        cacheFolderName,
        book.getFolderName(),
        bookChapter.getFileName(),
    ).writeText(content)
}
```

**优点：**
- 文本格式，便于直接读取
- 文件名使用MD5编码，避免特殊字符问题
- 章节序号前缀，便于排序和查找

## 四、自动清理机制

### 4.1 应用启动时清理

每次应用启动会调用 `BookHelp.clearInvalidCache()`，清理内容：

- 已从书架删除的书籍的缓存文件夹
- 已删除书籍的 epub 解压缓存
- 临时文件

**代码位置：** `App.kt`

```kotlin
BookHelp.clearInvalidCache()
```

### 4.2 漫画图片自动清理

针对**漫画类书籍**有专门的清理逻辑：

**清理规则：**
- 只清理漫画类书籍的图片
- 根据「图片保留数量」设置（默认为 0，即不清理）
- 保留当前章节前 N 章和后预下载数量的图片
- 超出范围的图片会被删除

**代码位置：** `BookHelp.kt`

```kotlin
private fun clearComicCache(book: Book) {
    // 只处理漫画，为0时不清除
    if (!book.isImage || AppConfig.imageRetainNum == 0) {
        return
    }
    // 向前保留设定数量，向后保留预下载数量
    val startIndex = book.durChapterIndex - AppConfig.imageRetainNum
    val endIndex = book.durChapterIndex + AppConfig.preDownloadNum
    // ...删除范围外的图片
}
```

### 4.3 普通文本章节不会被自动清理

**重要：普通小说的文本章节缓存不会被自动清空！**

- 文本章节内容（`.nb` 文件）会一直保留
- 除非手动清理或删除书籍

### 4.4 清理机制总结

| 缓存类型 | 是否自动清理 | 触发条件 |
|----------|--------------|----------|
| 已删除书籍的缓存 | ✅ 是 | 应用启动时 |
| 漫画图片 | ✅ 是（可配置） | 应用启动时，根据设置 |
| 普通文本章节 | ❌ 否 | 仅手动清理 |

### 4.5 相关设置

在 **设置 → 其他设置** 中：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 图片保留数量 | 漫画已读章节图片保留数量，0 表示不清理 | 0 |
| 预下载章节数 | 阅读时预下载的章节数量 | 0 |

### 4.6 手动清理方式

1. **我的 → 清除缓存** - 清除所有书籍缓存
2. **书籍详情页 → 清除缓存** - 清除单本书缓存
3. **书架管理 → 批量管理 → 清除缓存** - 批量清除

## 五、缓存备份可行性分析

### 5.1 现有备份机制

当前备份系统（`Backup.kt`）**不支持书籍缓存备份**。

**现有备份内容：**

| 类型 | 内容 |
|------|------|
| 数据库数据 | 书架、书签、书源、RSS源、替换规则、阅读记录等 |
| 配置文件 | 阅读配置、主题配置、封面规则等 |
| SharedPreferences | 应用设置 |
| 背景图片 | 阅读背景、主题背景 |

**不包含：** `book_cache` 目录下的章节缓存文件

### 5.2 备份方案设计

#### 方案一：打包整个 book_cache 目录

**实现方式：**
- 直接将整个缓存目录打包成 ZIP
- 可选择备份到本地或 WebDav

**优点：**
- 实现简单直接
- 完整保留所有缓存

**缺点：**
- 文件可能很大（几百MB甚至几GB）
- 备份和恢复耗时较长

#### 方案二：选择性备份

**实现方式：**
- 只备份指定书籍的缓存
- 用户可选择要备份的书籍

**优点：**
- 节省存储空间
- 用户可控制备份内容

**缺点：**
- 需要额外的选择界面
- 实现相对复杂

#### 方案三：增量备份

**实现方式：**
- 记录每个章节的缓存状态
- 只备份新增/变更的章节
- 适合 WebDav 云备份

**优点：**
- 节省带宽和存储
- 适合云端同步

**缺点：**
- 实现复杂
- 需要维护状态记录

### 5.3 需要考虑的问题

| 问题 | 说明 |
|------|------|
| 文件大小 | 缓存可能很大，影响备份速度和存储空间 |
| 书源有效性 | 恢复缓存后，书源可能已失效或更新 |
| 版本兼容 | 缓存格式可能随版本变化 |
| 存储位置 | Android 10+ 分区存储限制 |

### 5.4 实现思路

如果要实现缓存备份功能，大致流程如下：

**备份流程：**
```
1. 遍历 book_cache 目录
2. 读取书籍信息（从数据库匹配）
3. 打包章节文件 (.nb) 和图片
4. 生成缓存索引文件（记录书籍和章节对应关系）
5. 压缩并保存到本地或云端
```

**恢复流程：**
```
1. 解压备份文件
2. 读取缓存索引
3. 验证书籍是否存在于书架
4. 复制缓存文件到对应目录
5. 更新缓存状态
```

### 5.5 建议

1. **如果只是想保留阅读内容**：可以考虑使用现有的「导出」功能，导出为 TXT/EPUB 格式
2. **如果需要离线阅读**：使用现有的「缓存」功能手动下载
3. **如果确实需要备份缓存**：可以考虑新增一个缓存备份功能模块
