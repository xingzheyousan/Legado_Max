# 正文下一页懒加载与缓存机制

## 概述

正文下一页懒加载是一种按需加载策略，仅在用户阅读到一定进度时才预加载下一页内容。相比传统模式一次性加载所有分页，懒加载可以显著减少首次加载时间，节省流量。

## 配置

### 书源配置

在书源中设置 `nextPageLazyLoad` 字段：

```kotlin
// BookSource.kt
@ColumnInfo(defaultValue = "0")
var nextPageLazyLoad: Boolean = false  // 下一页懒加载
```

| 值 | 说明 |
|----|------|
| `true` | 启用懒加载，仅加载第一页，后续分页按需预加载 |
| `false` | 传统模式，一次性加载所有分页（默认） |

### 并发控制

懒加载采用**串行**加载策略：
- 每次只预加载一页
- 不受 `AppConfig.threadCount` 并发率控制
- 原因：后台预加载，用户无感知，串行更稳定省电

---

## ⚠️ 调试注意事项

**开启懒加载后，在书源调试界面不会访问正文下一页！**

原因：书源调试只调用一次内容获取，不会触发预加载逻辑。

**验证方法**：必须进入阅读界面，滚动到 70% 位置，观察是否触发下一页加载。

## 工作流程

### 传统模式

```
CacheBook 缓存 / 阅读加载
    ↓
WebBook.getContentAwait()
    ↓
一次性加载所有分页
    ↓
全部内容返回
```

### 懒加载模式

```
阅读加载（非缓存场景）
    ↓
WebBook.getContentLazyAwait()
    ↓
BookContent.analyzeContentLazy()  ← 只分析第一页
    ↓
返回 Pair(第一页内容, LazyContentManager)
    ↓
ReadBook.contentLoadFinishLazy()
    ↓
设置 lazyContent 到 TextChapter
    ↓
显示阅读界面
    ↓
用户滚动到 70%
    ↓
ScrollPageDelegate.checkLazyPrefetch()
    ↓
lazyContent.prefetchNextPage()  ← 预加载下一页
    ↓
TextChapter.appendContent()  ← 追加新内容
```

## 缓存与懒加载的关系

### 核心原则

**缓存时禁用懒加载，阅读时启用懒加载**

原因：
1. 懒加载的价值是"阅读时按需加载"，而不是"缓存时按需加载"
2. 缓存的目的就是保存完整内容
3. 架构简单，不需要改动缓存逻辑

### 场景判断

```kotlin
val useLazyLoad = bookSource.nextPageLazyLoad && isReadingMode
if (useLazyLoad) {
    // 走懒加载
} else {
    // 走传统模式，完整加载
}
```

| 场景 | `isReadingMode` | 行为 |
|------|-----------------|------|
| 缓存下载 | `false` | 传统模式，完整加载所有分页 |
| 阅读加载 | `true` | 懒加载模式（如果书源启用） |

## 架构实现

### 涉及的文件

| 文件 | 作用 |
|------|------|
| `LazyContentManager.kt` | 管理分页加载、预加载队列 |
| `BookContent.kt` | `analyzeContentLazy()` 分析懒加载内容 |
| `WebBook.kt` | `getContentLazyAwait()` 懒加载入口 |
| `ReadBook.kt` | `contentLoadFinishLazy()` 关联 lazyContent |
| `TextChapter.kt` | 持有 lazyContent，暴露 `appendContent()` |
| `TextChapterLayout.kt` | `appendContent()` 动态追加内容到章节 |
| `ScrollPageDelegate.kt` | `checkLazyPrefetch()` 触发预加载 |
| `CacheBook.kt` | 缓存逻辑，判断是否走懒加载 |

### 核心类：LazyContentManager

```kotlin
class LazyContentManager(
    private val bookSource: BookSource,
    private val book: Book,
    private val bookChapter: BookChapter,
    private val nextPageUrlList: List<String>
) {
    // 当前加载到的页码
    private var currentPageIndex: Int = 0
    
    // 预加载下一页
    suspend fun prefetchNextPage(): String?
    
    // 是否还有下一页
    fun hasNextPage(): Boolean
}
```

### 预加载触发

```kotlin
// ScrollPageDelegate.kt
fun checkLazyPrefetch() {
    val lazyContent = textChapter.lazyContent ?: return
    if (scrollProgress >= 0.7f && lazyContent.hasNextPage()) {
        // 滚动到 70% 时触发预加载
        scope.launch {
            lazyContent.prefetchNextPage()
        }
    }
}
```

### 内容追加

```kotlin
// TextChapter.kt
fun appendContent(newContent: String) {
    // 追加新内容到现有章节
    this.content += newContent
    // 通知 TextChapterLayout 重新排版
    contentChanged = true
}
```

## 与目录渐进加载的区别

| 特性 | 正文懒加载 | 目录渐进加载 |
|------|-----------|-------------|
| 配置位置 | 书源 `nextPageLazyLoad` | 全局 `isTocPartialLoad` |
| 触发条件 | 滚动到 70% | 立即开始 |
| 加载方式 | 按需预加载 | 边加载边展示 |
| 缓存行为 | 缓存时禁用 | 缓存时也生效 |
| 调试可见 | ❌ 不可见 | ✅ 可见 |
