# 目录不完全加载（渐进式加载）机制

## 概述

目录不完全加载是一种渐进式目录获取策略，允许在目录未完全加载时就展示已解析的章节，实现"边加载边展示"的用户体验。用户可以立即看到部分目录并开始阅读，而无需等待所有目录页加载完成。

## 核心配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `AppConfig.isTocPartialLoad` | 是否启用目录渐进加载 | - |
| `AppConfig.tocPartialLoadInterval` | 每页加载间隔（秒），用于控制请求频率 | 0 |

配置入口：`OtherConfigFragment.kt`

data class PartialChapterList(

## 加载流程

### 传统模式 vs 渐进模式

```
传统模式：
WebBook.getChapterListAwait()
    ↓
等待所有目录页加载完成
    ↓
一次性返回完整目录
    ↓
用户等待时间较长

渐进模式：
WebBook.getChapterListFlow()
    ↓
Flow 逐页发射 PartialChapterList
    ↓
每加载完一页立即通知 UI
    ↓
用户可立即看到部分目录
```

### 渐进加载详细流程

```
BookInfoViewModel.loadChapter()
    ↓
判断 isTocPartialLoad
    ↓ true
WebBook.getChapterListFlow()
    ↓
BookChapterList.analyzeChapterListFlow()
    ↓
解析第一页目录
    ↓
emit PartialChapterList(chapters, isComplete=false)
    ↓
ViewModel 收集 Flow
    ↓
chapterListData.postValue(chapters)  ← UI 立即刷新
    ↓
savePartialBookChapters(chapters)    ← 增量保存到数据库
    ↓
继续加载下一页...
    ↓
emit PartialChapterList(allChapters, isComplete=true)
    ↓
replaceBookChapters()                ← 最终替换完整目录
```

## 加载策略

BookChapterList 根据目录页数采用不同策略：

| 场景 | 策略 | 说明 |
|------|------|------|
| 单页目录 | 直接发射 | 只发射一次 `isComplete=true` |
| 串行多页（1个nextUrl） | 逐页发射 | 每加载完一页发射一次，最后一页 `isComplete=true` |
| 并发多页（多个nextUrl） | 增量发射 | 每完成一个请求发射一次中间结果，全部完成后发射 `isComplete=true` |

### 串行多页示例

```
第1页加载完成 → emit(chapters1, false)
第2页加载完成 → emit(chapters1+2, false)
第3页加载完成 → emit(chapters1+2+3, true)
```

### 并发多页示例

```
同时请求第2、3、4页
第2页返回 → emit(chapters1+2, false)
第4页返回 → emit(chapters1+2+4, false)
第3页返回 → emit(chapters1+2+3+4, true)
```

## 请求间隔控制

为避免对服务器造成过大压力，渐进加载模式下会控制请求间隔：

```kotlin
private suspend fun delayBeforeNextTocPage(bookSource: BookSource) {
    if (!AppConfig.isTocPartialLoad) {
        return  // 非渐进模式不延迟
    }
    val interval = AppConfig.tocPartialLoadInterval.coerceIn(0, 60) * 1000L +
            parseConcurrentRateInterval(bookSource.concurrentRate)
    if (interval > 0) {
        delay(interval)
    }
}
```

间隔 = 用户配置间隔 + 书源并发率间隔

## 并发控制

渐进加载模式下，多页目录的并发请求受到限制：

```kotlin
.mapAsync(if (AppConfig.isTocPartialLoad) 1 else AppConfig.threadCount) { urlStr ->
    // 请求处理
}
```

| 模式 | 并发数 | 说明 |
|------|--------|------|
| 渐进加载 | 1 | 串行请求，逐页加载 |
| 传统模式 | `AppConfig.threadCount` | 并发请求，提高速度 |

## UI 层处理

### ChapterListFragment

```kotlin
// 判断是否为渐进加载的书籍
val isPartialLoadingBook = AppConfig.isTocPartialLoad &&
        currentBook?.isLocal == false &&
        !currentBook.readSimulating()

// 渐进加载时，查询范围设为 Int.MAX_VALUE
val end = if (isPartialLoadingBook) {
    Int.MAX_VALUE
} else {
    currentBook.simulatedTotalChapterNum() - 1
}

// 从数据库查询已加载的章节
appDb.bookChapterDao.getChapterList(bookUrl, 0, end)
```

### 加载状态显示

```kotlin
// 监听目录加载完成事件
observeEventSticky<String>(EventBus.TOC_LOAD_COMPLETE) { bookUrl ->
    if (viewModel.bookUrl == bookUrl) {
        binding.tvTocLoading.gone()  // 隐藏加载提示
    }
}
```

## 数据库操作

### 增量保存

```kotlin
private fun savePartialBookChapters(chapters: List<BookChapter>) {
    // 增量保存，不删除已有数据
    appDb.bookChapterDao.insert(*chapters.toTypedArray())
}
```

### 最终替换

```kotlin
private fun replaceBookChapters(oldBook: Book, chapters: List<BookChapter>) {
    // 删除旧数据，插入完整目录
    appDb.bookChapterDao.delByBook(oldBook.bookUrl)
    appDb.bookChapterDao.insert(*chapters.toTypedArray())
}
```

## 事件通知

| 事件 | 触发时机 | 用途 |
|------|----------|------|
| `EventBus.TOC_PARTIAL_LOADED` | 每次发射中间结果 | 通知目录增量更新 |
| `EventBus.TOC_LOAD_COMPLETE` | Flow 收集完成 | 隐藏加载提示 |

## 与阅读流程的集成

渐进加载允许用户在目录未完全加载时就开始阅读：

```kotlin
// 中间过程：增量更新，不触发内容加载

// 最终完成：正常更新
ReadBook.onChapterListUpdated(book)
```

## 使用场景

### 适用场景
- 目录页数较多（如分页目录网站）
- 用户希望快速开始阅读
- 网络较慢时提升体验

### 不适用场景
- 本地书籍（`book.isLocal == true`）
- 模拟阅读模式（`book.readSimulating() == true`）
