# 书架和订阅下拉刷新流程与机制

## 概述

app的下拉刷新基于Android官方组件`SwipeRefreshLayout`实现，核心流程为：**用户下拉 → 触发刷新监听 → 执行业务刷新逻辑 → 隐藏刷新动画**。

## 二、初始化流程

所有下拉刷新的初始化包含三个关键步骤：

### 步骤1：设置刷新颜色
```kotlin
binding.refreshLayout.setColorSchemeColors(accentColor)
```

### 步骤2：设置刷新监听
```kotlin
binding.refreshLayout.setOnRefreshListener {
    binding.refreshLayout.isRefreshing = false  // 立即隐藏动画
    activityViewModel.upToc(books, onlyUpdateRead) // 执行刷新逻辑
}
```

### 步骤3：控制刷新开关
```kotlin
// 空列表时禁用下拉刷新
binding.refreshLayout.isEnabled = enableRefresh && itemCount > 0
```

| `setColorSchemeColors()` | 刷新动画颜色 |

## 四、书架下拉刷新机制

### 完整流程图

```
用户下拉触发
    ↓
【立即隐藏刷新动画】
    ↓
【调用 MainViewModel.upToc()】
    ├─ 过滤本地书籍和不可更新的书籍
    ├─ 添加到等待队列 waitUpTocBooks
    └─ 启动 startUpTocJob()（若未运行）
    ↓
【startUpTocJob 并发处理】
    ├─ 从队列取出 bookUrl
    ├─ 发送 UP_BOOKSHELF 事件（刷新前）
    ├─ 执行 updateToc(bookUrl)
    └─ 发送 UP_BOOKSHELF 事件（刷新后）
    ↓
【updateToc 单本书更新】
    ├─ 获取书籍和书源
    ├─ 书源信号量控制（同书源串行）
    ├─ WebBook.getChapterListAwait() 获取目录
    ├─ 更新数据库（书籍+章节）
    ├─ 触发 ReadBook.onChapterListUpdated()
    └─ 添加预下载任务
    ↓
【UI层响应事件】
    ├─ Flow 自动发射新数据
    ├─ Adapter 更新列表
    └─ 刷新完成
```

[MainViewModel.upToc](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/ui/main/MainViewModel.kt#L127-L135)
```kotlin
fun upToc(books: List<Book>, onlyUpdateRead: Boolean) {
    execute(context = upTocPool) {
        books.filter { !it.isLocal && it.canUpdate }
            .let { addToWaitUp(it, onlyUpdateRead) }
    }
}
```

[startUpTocJob](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/ui/main/MainViewModel.kt#L156-L188)
```kotlin
private fun startUpTocJob() {
    upTocJob = viewModelScope.launch(upTocPool) {
        flow {
            while (true) emit(waitUpTocBooks.poll() ?: break)
        }.onEachParallel(threadCount) { bookUrl ->
            onUpTocBooks.add(bookUrl)
            postEvent(EventBus.UP_BOOKSHELF, bookUrl)  // 刷新前事件
            updateToc(bookUrl)
        }.onEach { bookUrl ->
            onUpTocBooks.remove(bookUrl)
            postEvent(EventBus.UP_BOOKSHELF, bookUrl)  // 刷新后事件
            postUpBooksLiveData()
        }.onCompletion {
            upTocJob = null
            if (waitUpTocBooks.isNotEmpty()) startUpTocJob()  // 递归继续
        }.collect()
    }
}
```

## 五、RSS文章下拉刷新机制

RSS文章页的刷新机制略有不同，采用**网络请求 + 数据库更新**模式。

### 完整流程图

```
用户下拉触发
    ↓
【调用 loadArticles()】
    ↓
【ViewModel.loadArticles()】
    ├─ 网络请求获取文章列表
    ├─ 保存到数据库 rssArticleDao
    └─ 设置 fullRefresh = true
    ↓
【数据库Flow发射】
    ├─ fullRefresh 时全量更新
    ├─ 否则使用 DiffUtil 差异化更新
    └─ Adapter.setItems()
    ↓
【隐藏刷新动画】
    refreshLayout.isRefreshing = false
```

## 七、关键设计要点

### 1. 立即隐藏刷新动画

所有下拉刷新都在回调第一行立即设置`isRefreshing = false`，而不是等待刷新完成：

```kotlin
binding.refreshLayout.setOnRefreshListener {
    binding.refreshLayout.isRefreshing = false  // 立即隐藏
    executeRefreshLogic()  // 异步执行刷新
}
```

**原因**：刷新逻辑是异步的（网络请求、数据库操作），立即隐藏避免动画长时间停留。

### 2. 并发控制

书架刷新使用**书源信号量**控制并发：

```kotlin
val semaphore = sourceSemaphores.getOrPut(source.bookSourceUrl) { Semaphore(1) }
semaphore.acquire()  // 同书源串行执行
try {
    updateToc(bookUrl)
} finally {
    semaphore.release()
}
```

### 3. 事件通知机制

刷新前后通过EventBus发送事件，UI层监听并更新：

```kotlin
// ViewModel发送
postEvent(EventBus.UP_BOOKSHELF, bookUrl)

// Fragment监听
observeEvent<String>(EventBus.UP_BOOKSHELF) {
    booksAdapter.notification(it)
}
```

### 4. Flow自动更新

书架数据使用Flow响应式更新：

```kotlin
appDb.bookDao.flowByGroup(groupId)
    .flowWithLifecycleAndDatabaseChangeFirst(...)
    .collect { list ->
        booksAdapter.setItems(list)  // 自动刷新UI
    }
```

### 5. DiffUtil差异化更新

RSS文章使用DiffUtil优化更新性能：

```kotlin
adapter.setItems(newList, object : DiffUtil.ItemCallback<RssArticle>() {
    override fun areItemsTheSame(old: RssArticle, new: RssArticle) = 
        old.link == new.link
    
    override fun areContentsTheSame(old: RssArticle, new: RssArticle) = 
        old.title == new.title && old.image == new.image
})
```

### 6. 刷新开关控制

根据不同场景动态启用/禁用下拉刷新：

```kotlin
// 空列表禁用
binding.refreshLayout.isEnabled = enableRefresh && itemCount > 0

// 分组级别控制
binding.refreshLayout.isEnabled = it.enableRefresh

// 拖拽时禁用
swipeRefreshLayout?.isEnabled = !swiping
```

### 7. 预加载自动刷新

RSS文章支持预加载时自动触发刷新：

```kotlin
if (isPreload) {
    refreshLayout.post {
        refreshLayout.isRefreshing = true
        loadArticles()
    }
}
```

## 八、总结

下拉刷新的核心流程：

```
布局包裹SwipeRefreshLayout
    ↓
初始化：setColorSchemeColors + setOnRefreshListener
    ↓
用户下拉 → 触发监听 → 立即隐藏动画
    ↓
异步执行业务逻辑（网络/数据库）
    ↓
Flow/EventBus通知UI更新
    ↓
刷新完成
```

关键设计原则：
1. **立即隐藏动画**，避免长时间等待
2. **异步执行刷新**，不阻塞UI线程
3. **响应式更新UI**，通过Flow/EventBus自动刷新
4. **并发控制**，通过信号量限制同书源并发
5. **差异化更新**，通过DiffUtil减少UI重绘
