# 书籍缓存导出 ZIP 恢复修复方案

## 问题现象

从“书籍缓存选择”界面单独导出某些书的缓存 ZIP 后，如果用户重置软件数据，再通过“恢复”功能选择刚导出的缓存 ZIP，恢复后书架上有时看不到对应书籍，缓存也就无法正常使用。

这个 ZIP 不是常规完整备份 ZIP。它原本主要包含章节缓存文件和缓存索引，缺少恢复到空数据库时所需的书架上下文。

## 根因

恢复书籍缓存不是简单把文件复制回 `book_cache` 目录。阅读缓存能被使用，需要同时满足几个条件：

1. `books` 表里必须存在对应书籍。
2. `book_chapters` 表里必须存在对应章节目录。
3. 缓存文件必须位于 `book_cache/<书籍缓存目录>/` 下。
4. 书籍必须能被书架查询查到。
5. 重置数据后默认书籍分组必须存在，否则书架根列表的 SQL 可能过滤掉已插入的书。

原问题主要有三层：

- 单独导出的缓存 ZIP 没有携带完整书籍记录。重置数据后，恢复端无法在空书架中找到目标书。
- 旧导出结构可能把每本书缓存目录直接放在 ZIP 根目录，而恢复端期望 `book_cache/<folderName>`。
- 即使恢复端补插了 `Book`，重置数据后 `book_groups` 默认分组可能未初始化，书架根查询依赖“网络未分组”等默认分组，导致书已在数据库但书架不显示。

## 最终方案

### 1. 导出缓存 ZIP 时携带书籍清单

在书籍缓存选择界面导出 ZIP 时，除原有内容外新增：

- `bookCacheBooks.json`：保存被导出缓存对应的 `Book` 列表。

导出的 ZIP 现在应包含：

```text
book_cache/
  <book-folder>/
    *.nb
    images/
bookCacheIndex.json
bookChapterCache.json
bookCacheBooks.json
```

其中：

- `bookCacheIndex.json` 用于缓存目录和章节缓存文件匹配。
- `bookChapterCache.json` 用于恢复章节目录。
- `bookCacheBooks.json` 用于重置数据后重建书架书籍记录。

### 2. 恢复缓存前先补书架书籍

恢复 `book_cache` 前，恢复端会先读取 `bookCacheBooks.json`，将缺失书籍插入 `books` 表。

如果恢复的是旧缓存 ZIP，没有 `bookCacheBooks.json`，则使用 `bookCacheIndex.json` 生成最小可用的 `Book` 记录作为兜底。

补插书籍时会处理可见性：

- `group = 0`，避免引用重置后不存在的自定义分组。
- 移除 `BookType.notShelf` 标记，确保书籍属于正式书架。

### 3. 先补默认书籍分组

恢复缓存书籍前会确保默认 `book_groups` 存在，包括：

- 全部
- 本地
- 音频
- 网络未分组
- 本地未分组
- 视频
- 更新失败

这是书架列表能显示恢复书籍的关键。书架根列表查询依赖这些默认分组的 `show` 状态，尤其是“网络未分组”。

### 4. 恢复章节目录和缓存文件

补完书籍后，再恢复 `bookChapterCache.json`。

随后根据 `bookCacheIndex.json` 匹配书籍和章节，将缓存文件复制到目标书籍缓存目录：

```text
BookHelp.cachePath/<restored-book-folder>/
```

章节缓存文件会按当前章节的 `getFileName()` 重新落位，避免不同设备或不同书籍 URL 下文件名不一致导致读不到缓存。

### 5. 兼容旧 ZIP 结构

恢复端同时兼容两种缓存目录结构：

```text
book_cache/<folderName>/
```

以及旧导出可能出现的：

```text
<folderName>/
```

如果找不到 `book_cache/`，恢复端会按 `bookCacheIndex.json` 里的 `folderName` 在 ZIP 根目录查找对应缓存目录。

## 恢复流程

实际恢复顺序如下：

1. 解压用户选择的缓存 ZIP。
2. 读取 `bookCacheIndex.json`。
3. 确保默认书籍分组存在。
4. 读取 `bookCacheBooks.json`，或用缓存索引生成最小书籍记录。
5. 插入缺失书籍，并保证书籍可在书架显示。
6. 恢复 `bookChapterCache.json` 到章节目录表。
7. 查找 `book_cache/` 或旧 ZIP 根目录缓存。
8. 将章节缓存文件复制到当前书籍缓存目录。
9. 恢复完成后书架可看到对应书籍，阅读时可直接命中章节缓存。

## 涉及文件

- `app/src/main/java/io/legado/app/ui/book/cacheSelector/BookCacheSelectorViewModel.kt`
  - 单独导出书籍缓存 ZIP 时新增 `bookCacheBooks.json`。
  - 将缓存目录按 `book_cache/` 结构整体打包。

- `app/src/main/java/io/legado/app/help/storage/Restore.kt`
  - 识别 `bookCacheBooks.json`。
  - 恢复缓存前补书籍记录。
  - 补齐默认书籍分组。
  - 兼容旧 ZIP 根目录散放缓存目录。

- `app/src/main/java/io/legado/app/help/storage/Backup.kt`
  - 常规备份中的书籍缓存也同步写入 `bookCacheBooks.json`。

- `app/src/main/java/io/legado/app/help/storage/BackupInfoHelper.kt`
  - 将 `bookCacheBooks.json` 纳入备份/恢复信息展示和书籍缓存选择规则。
